Git & GitHub

Conceptos clave *

~ Maryam El Harrak Talibi ~

En el contexto del desarrollo de aplicaciones web, GitHub se presenta como una herramienta esencial
para el control de versiones y la colaboración en equipo. Primero, es importante entender que Git es
el sistema que permite gestionar el historial de cambios en el código, mientras que GitHub actúa como
una plataforma en la nube para almacenar esos repositorios, facilitando el trabajo en equipo.

¿Qué es lo más importante con lo que nos debemos quedar?

_________________________________________________________ Repositorios ____________________________________________________________

Son el núcleo de cualquier proyecto en GitHub, donde se guarda tanto el código como su historial.
La creación de un repositorio en GitHub es simple y puede hacerse sin necesidad de un archivo README.

__________________________________________________________ Ramas ________________________________________________________________

El manejo de ramas es fundamental para el flujo de trabajo. Permiten a los desarrolladores trabajar en
características nuevas o correcciones sin afectar la rama principal. Esto es crucial para mantener la estabilidad
del proyecto mientras se implementan mejoras.

______________________________________________________ Fusión y Conflictos _________________________________________________________

Al combinar ramas, es común enfrentar conflictos, especialmente si se han hecho cambios en las mismas
líneas de un archivo. El artículo explica cómo resolver estos conflictos mediante la edición de archivos,
asegurando que el commit final sea limpio y ordenado.

________________________________________________________ Etiquetas o Tags _________________________________________________________

Se utilizan para marcar versiones específicas del proyecto,
lo que resulta muy útil en la gestión de lanzamientos.


En conclusión...

El uso efectivo de Git y GitHub no solo mejora la organización del código, sino que también permite una colaboración más fluida
entre los miembros del equipo. Dominar estas herramientas es vital para cualquier desarrollador en el campo actual, donde el trabajo en equipo
y la gestión del código son aspectos cruciales del desarrollo de software.
